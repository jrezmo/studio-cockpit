Version 2023.09

Compatibility

PTSL SDK 2023.09 uses PTSL_VERSION 1 of the client interface

Pro Tools 2023.9 is compatible with clients using PTSL_VERSION 1 and higher

Reminder: Developer certificates have been deprecated

Pro Tools 2023.9 is the last release to support AuthorizeConnection

The AuthorizeConnection API has been superseded by the RegisterConnection API, introduced in PT 2023.3. This new method no longer requires a certificate to authenticate with Pro Tools upon connection, and simply requires you provide the "companyName" and "applicationName" when sending commands from the SDK.

For more details, see RegisterConnectionRequest and RegisterConnectionResponse.

Changelog

General changes

  • grpc for both C++ client and Pro Tools server is updated to 1.43.0 additionally with its dependencies (previously was 1.37.0)
  • protobuf for both C++ client and Pro Tools server is set to version 3.19.2 (previously was unspecified)

Protocol changes

New commands:

  • CreateNewTracks
  • SelectTracksByName
  • GetEditMode
  • SetEditMode
  • GetEditTool
  • SetEditTool
  • RecallZoomPreset
  • GetEditModeOptions
  • SetEditModeOptions
  • SetTimelineSelection
  • GetTimelineSelection

New possible errors:

  • PT_Forbidden
  • PT_NoTimelineFound
  • PT_ArgumentOutOfRange

New enumerations:

  • TrackFormat
  • TrackTimebase
  • EditMode
  • EditTool
  • TimelineUpdateVideo
  • SelectionMode

New messages:

  • PaginationRequest
  • PaginationResponse
  • CreateNewTracksRequestBody and CreateNewTracksResponseBody
  • EditModeOptions
  • GetEditModeResponseBody
  • SetEditModeRequestBody
  • GetEditToolResponseBody
  • SetEditToolRequestBody
  • RecallZoomPresetRequestBody
  • SetEditModeOptionsRequestBody
  • GetEditModeOptionsResponseBody
  • SetTimelineSelectionRequestBody
  • GetTimelineSelectionRequestBody and GetTimelineSelectionResponseBody
  • SelectTracksByNameRequestBody and SelectTracksByNameResponseBody

Existing command changes:

  • TrackType enumeration: added prefixes for values, non prefix values are deprecated
  • Pagination message is deprecated. Use PaginationRequest instead
  • Changes in AudioData message
    • destination is deprecated. Use audio_destination instead
    • location is deprecated. Use audio_location instead
  • Changes in ImportResponseBody message
    • file_list, audio_operations, destination_path are deprecated. Use audio_data instead
    • The response is not fully implemented yet. The values are not expected to be returned at the moment.
  • Changes in GetTrackListRequestBody message
    • page_limit is deprecated. Use pagination_request instead
  • Changes in GetTrackListResponseBody message
    • stats is deprecated. Use pagination_response instead
  • Changes in GetFileLocationRequestBody message
    • page_limit is deprecated. Use pagination_request instead
  • Changes in GetFileLocationResponseBody message
    • stats is deprecated. Use pagination_response instead

C++ client changes

  • CMake 3.25.0 is the minimum requirement (previously was unspecified). The check is not enforced but it's not guaranteed to be compatible with older versions
  • C++17 is the minimum requirement (previously was C++14)
  • -arch armv8 option is deprecated in build_cpp_ptsl_sdk.py. Use arm64 instead
  • -arch x86_64_arm64 (universal binary) option is now the default option for MacOS in build_cpp_ptsl_sdk.py
  • find_package for the C++ client supports PTSLC_CPP_DEBUG_ROOT/PTSLC_CPP_RELEASE_ROOT CMake variables to hint additional search path in multi-config projects
  • Improved universal binary/cross-compilation (architecture) support for the C++ client
  • CreateFade is deprecated. Use CreateFadesBasedOnPreset
  • RenameTrack is deprecated. Use RenameTargetTrack
