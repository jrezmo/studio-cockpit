Compatibility

PTSL SDK 2025.06 uses:

  • Old format:
    • PTSL_VERSION 2025 of the client interface
  • New format:
    • Major Version 2025
    • Minor Version 6
    • Revision Version 0

Pro Tools 2025.06 is compatible with clients using Major Version 1 and higher.

For further information, refer to the Versioning Changes sections.

Changelog

Versioning Changes

Version Triplet

Starting with the 2025.06 release, the PTSL version has been updated from 5 (2024.10) to 2025 (2025.XX). Additionally, version_minor (6) and version_revision (0) are now used to differentiate releases and subreleases within the year.

Old clients (prior to this release) will receive only the major version (2025) for all Pro Tools releases in 2025.

New clients (starting from this release) can process the entire triplet (version, version_minor, version_revision) to differentiate between Pro Tools releases. Older Pro Tools releases only have the major version, i.e., 5.0.0 for 2024.10.

Protocol

Starting with the 2025.06 release, the SDK package contains multiple .proto files. Each file includes the full protocol description for a release. For example, PTSL.2024.10.0.proto contains the protocol definition for the 2024.10 release, wrapped inside package ptsl_2024_10_0. The current release is always named PTSL.proto, wrapped inside package ptsl.

Using the SendRequest and CppPTSLRequest, it is possible to send requests from different releases to the same Host. The Host response is adapted to the capabilities and protocol of the release requested by the client.

There are two known limitations to this:

  • Host-side versioning is an area of active work:
    • Some deprecated enumeration values may be prioritized over their new replacements regardless of the request version.
    • Some commands may return both old properties and their new replacements.
  • Host-side versioning is a relatively new addition:
    • It is not always possible to request API behavior similar to an older release, e.g., 2023.03.

C++ Client

The C++ client reflects the changes mentioned above. Both CppPTSLRequest and CppPTSLResponse provide the ability to get and set the version triplet.

Old command-specific methods are now pinned to the 2024.10 PTSL version and are no longer expected to be updated.

Enumeration Changes

There are several changes to enumerations to reduce name clashes and make their behavior more uniform:

  • All enumeration values now have a prefix based on their type name.
    • The old values are deprecated but still accepted and can be returned by the Host.
  • All enumeration types (except CommandId) now start with the '*_Unknown value = 0'.
    • This may offset values in some types, but it will not affect communication with the Host as only the string names are used.

Intermediate Responses

This release enhances support for intermediate responses during command execution. A single task can now emit multiple InProgress intermediate responses, providing improved progress tracking and feedback. These improvements are applied universally to all clients from the Pro Tools side, enabling even older SDK versions to process them if desired. While intermediate response capability existed in the very first SDK release, the 2025.06 update delivers Pro Tools-side enhancements that improve functionality across all client implementations. New clients built with the updated SDK can now process intermediate responses seamlessly via updated SendRequest.

Key updates:

  • Intermediate responses may include a JSON body, typically reserved for specific APIs such as the Event System (see below).
  • The final response consolidates all accumulated data, including errors, warnings, and informational messages generated during execution.

C++ Client:

  • The SendRequest method has been updated to include a new callback mechanism. This enables clients to receive all responses associated with a task, including intermediate responses, in real-time.

Event System (Experimental)

The 2025.06 release introduces the Event System, enabling Pro Tools to send real-time notifications to subscribed clients. This system allows clients to subscribe to specific events, unsubscribe from them, and receive event notifications as they occur.

Workflow

  1. SubscribeToEvents:
    • Adds specified events to the subscription list for the current PTSL session.
    • Each subscription is uniquely identified by the combination of PTSL session, EventId, and filter data. Any difference in filter data results in a new subscription, which is managed independently, even if they share the same EventId.
    • For example, a hypothetical event EId_TrackRecordEnabledStateChanged could include a track ID in the filter data to listen for changes to the record-enabled state of a specific track.
  2. UnsubscribeFromEvents:
    • Removes specified events (and their associated filter data) from the subscription list for the current PTSL session.
  3. PollEvents:
    • A long-polling API that streams real-time events to the client.
    • Operates indefinitely without a typical end condition.
    • Sends events as they are reported by Pro Tools, using the Intermediate Responses mechanism.
    • Only one active PollEvents connection is allowed per PTSL session. If a new PollEvents call is made, the previous connection is terminated, and polling is transferred to the new connection.
    • All event-related APIs, including PollEvents, are processed with higher execution priority compared to normal API commands, ensuring timely delivery of events without blocking other API commands.

Supported Events in 2025.06

The Event System supports the following events:

  • SessionOpened: Triggered when Pro Tools has finished opening a session.
  • SessionCreated: Triggered when a new Pro Tools session has been created.
  • SessionClosed: Triggered when a current Pro Tools session has been closed.

Command Progress Tracking

With 2025.06 release, several commands have been enhanced to provide accurate progress tracking information. The host now reports precise command status updates as these commands execute. Furthermore, the updated commands can return a "Failed" status when an internal operation encounters an error.

Updated commands:

  • OpenSession
  • Cut
  • Copy
  • Paste
  • Clear
  • CutSpecial
  • CopySpecial
  • PasteSpecial
  • ClearSpecial
  • GroupClips
  • UngroupClips
  • UngroupAllClips
  • RegroupClips
  • Spot
  • CreateFadesBasedOnPreset

Please note that the command progress tracking infrastructure is implemented entirely on the host side. This means:

  • Products using older SDKs connecting to Pro Tools 2025.06 or newer will automatically benefit from command progress tracking without any code changes.
  • Products built with SDK version 2025.06 connecting to Pro Tools versions earlier than 2025.06 will not have improved command progress tracking functionality.
