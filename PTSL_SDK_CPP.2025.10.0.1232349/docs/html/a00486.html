Version 2024.06

Compatibility

PTSL SDK 2024.06 uses PTSL_VERSION 4 of the client interface

Pro Tools 2024.06 is compatible with clients using PTSL_VERSION 1 and higher

DEVELOPER CERTIFICATES ARE NO LONGER SUPPORTED

You no longer need to obtain a developer certificate to establish a PTSL connection to Pro Tools. Use the new RegisterConnection command to establish connections to Pro Tools 2023.03 and higher.

Existing command changes:

  • Changes in EM_AudioInfo message
    • sample_rate_custom field was added
  • Changes in CreateNewTracksRequestBody message
    • pagination_request field was added
    • insertion_point_position field was added
    • insertion_point_track_name field was added
  • Changes in CreateNewTracksResponseBody message
    • created_track_names field was added
    • pagination_response field was added
    • number_of_tracks field was deprecated
  • Changes in HostReadyCheck command
    • is_host_ready returns true after full Pro Tools initialization
  • Changes in ClearMemoryLocation command
    • success_count field was added
    • failure_count field was added
    • failure_list field was added

C++ client changes

  • HostReadyCheck command is now public and can be called directly
  • C++ client now supports "direct mode"
    • the new mode allows sending a JSON string (message request) alongside the command instead of filling C++ structure with the command arguments
    • see directJsonBody. Non-empty string enables the mode
  • C++ client now privately depends on the Howard Hinnant date library version 3.0.1. The library is not exposed in public headers
