Export Mix (Bounce Mix) can export an entire mix, including sub-mixes, to multiple file formats without having to create extra dedicated audio tracks in the session for bus recording. Different options and settings are available depending on the selected FileType. The options are organized in sections including Audio, Video, Location, and Dolby Atmos. Here's a breakdown of how the sections are used for each FileType:

  • WAV
    1. has no Video section
    2. has a unique field called DeliveryFormat (only shown if Interleaved is selected as the file format and there are multiple mix sources)
  • AIF
    1. has no Video section
  • MP3
    1. has no Video section
    2. does not allow multiple mix sources
    3. does not allow mix sources greater than 0
    4. does not allow specifying Bit Depth value (always a 16Bit)
    5. Sample Rate is limited to 48k and 44.1k
    6. Import after Bounce is not available
  • MXF OPAtom
    1. has no Video section
    2. File format is multi-mono
    3. Bit Depth is only 16Bit and 24Bit
    4. Sample Rate is limited to 48k and 44.1k
    5. Pad to frame boundary is hard-coded to ON

  • WAV (Dolby Atmos ADM BWF)

    1. has no Audio section
    2. has no Video section
    3. has Dolby Atmos section (unique to this format)
    4. has FFOA (bool) default to False
    5. if FFOA is true then there's an option to enter TimeCode value
      1. TimeCode value will match the session frame rate
      2. Frame rate will always be 23.976 and 30 non-drop (if Frame rate is 60 -> it will still be 30)
    6. has Embed Program-level Metadata
