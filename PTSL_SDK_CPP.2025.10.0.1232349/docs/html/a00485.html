Version 2024.03

Compatibility

PTSL SDK 2024.03 uses PTSL_VERSION 3 of the client interface

Pro Tools 2024.03 is compatible with clients using PTSL_VERSION 1 and higher

DEVELOPER CERTIFICATES ARE NO LONGER SUPPORTED

You no longer need to obtain a developer certificate to establish a PTSL connection to Pro Tools. Use the new RegisterConnection command to establish connections to Pro Tools 2023.3 and higher.

Changelog

Protocol changes

New command:

  • GetSessionIDs

New message:

  • GetSessionIDsResponseBody

Existing command changes:

  • Changes in TrackFormat enumeration
    • TF_None value was added
    • TF_2_1 value was added
    • TF_Overhead value was added
  • Changes in TrackTimebase enumeration
    • TTB_None value was added
  • Changes in TrackAttributes message
    • is_record_enabled_safe field was added
    • is_smart_dsp_on_safe field was added
    • is_soloed_safe field was added
  • Changes in Track message
    • format field was added
    • timebase field was added
  • Changes in ExportClipsAsFilesRequestBody message
    • sample_rate_custom field was added

C++ client changes

  • C++ client build script now supports -vs_compiler option to specify the Visual Studio compiler version
    • supported values: vs2017, vs2019, vs2022
  • C++ client was updated to support both CommandError and ResponseError messages from Pro Tools
