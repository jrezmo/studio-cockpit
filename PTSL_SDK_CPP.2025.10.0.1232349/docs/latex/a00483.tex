Pro Tools 2023.\+9 is compatible with clients using \mbox{\hyperlink{a00446_a83c5961efacc6c06e3e75877c239ebb1}{PTSL\+\_\+\+VERSION}} 1 and higher

Reminder\+: \mbox{\hyperlink{a00481_release_notes_2023_03_changelog_developer_certificates_have_been_deprecated}{Developer certificates have been deprecated}}

Pro Tools 2023.\+9 is the last release to support Authorize\+Connection

The Authorize\+Connection API has been superseded by the \mbox{\hyperlink{a01328_aca497935e1b825a7c9cd5e28e7c2080b}{Register\+Connection}} API, introduced in PT 2023.\+3. This new method no longer requires a certificate to authenticate with Pro Tools upon connection, and simply requires you provide the \char`\"{}company\+Name\char`\"{} and \char`\"{}application\+Name\char`\"{} when sending commands from the SDK.

For more details, see \mbox{\hyperlink{a01756}{Register\+Connection\+Request}} and \mbox{\hyperlink{a01760}{Register\+Connection\+Response}}.\hypertarget{a00483_release_notes_2023_09_changelog}{}\doxysubsection{Changelog}\label{a00483_release_notes_2023_09_changelog}
